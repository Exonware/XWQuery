# Double-check: Template migration (no HTML in TS)

**Date:** 2026-01-29  
**Scope:** All component testers under `src/components/*/testers/`.

## 1. No main `testArea.innerHTML` in TS

- **Check:** `testArea.innerHTML = \`` in any `**/testers/*.ts`
- **Result:** **0 matches** – no tester uses inline HTML for the main content.

## 2. Template ID consistency (TS ↔ HTML)

- **Check:** Every `getElementById('tester-...-content')` in TS has a matching `<template id="tester-...-content">` in the same-name HTML.
- **Result:** **193 TS** template IDs, **193 HTML** template IDs, **0 mismatches** (IDs in TS but not in HTML: none; IDs in HTML but not in TS: none).

## 3. External script only (no inline TS in HTML)

- **Check:** Tester HTML files use `<script type="module" src="./Tester*.ts">` only.
- **Result:** **218** tester/index HTML files use `script type="module" src=...`; no inline `<script>...</script>` with code in our Tester*.html or testers/index.html. (Inline script only in reference files like mui.html, sample.html.)

## 4. Testers index

- **index.html:** `<script type="module" src="./index.ts">` (no inline script).
- **index.ts:** Imports `components` from `./index-data`, then render/search logic.
- **index-data.ts:** Generated by `tools/generate-component-index.ts`, exports `components`.

## 5. Counts

| Item                         | Count |

## 6. Remaining `innerHTML` in testers (by design)

- **testers/index.ts:** `grid.innerHTML`, `stats.innerHTML` (small dynamic UI).
- **Some Tester*.ts:** Small fragments only – e.g. `controlsElement.innerHTML`, `div.innerHTML` in callbacks, error messages with `${error.message}`. See `docs/guides/XWUI_TESTER_CONVENTIONS.md` §3.

**Conclusion:** Migration is consistent: no main HTML in TS, 193 testers use template + clone, all tester HTML uses external script only, testers index uses index-data and no inline script.
